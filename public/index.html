What is DoubleSigma?

DoubleSigma is a deterministic codemod-style migrator for common ethers.js v5 → v6 API rewrites, with a quantum-inspired latent-pattern scan and an optional Groq advisory pass for leftovers (no auto-apply from AI — review notes in the log).

  • Providers, constants, utils renames, conservative BigNumber literals, narrow .deployed() cleanup, @ethersproject/* import paths (abi / providers / tagged bignumber)
  • GitHub zipball import — no local git required
  • Benchmarks tab: dry-run the curated public repos; export CSV/JSON/HTML for judges
How it works
Quantum scan

Read-only ranking + latent v5 markers (BigNumber, nested providers, …).

Codemods

String transforms from one catalog — same engine as CLI.

AI (Groq)

Set GROQ_API_KEY in .env; metrics: time, tokens, files reviewed.

Benchmarks

Run all / export — numbers are measured on your machine.

Quick start: (1) Choose a preset or paste a GitHub URL → (2) Run dry-run → (3) inspect diffs; use Apply only when satisfied.

Why DoubleSigma?

Positioning for judges: focused ethers v5→v6 codemods, real multi-repo benchmarks, deterministic core rules.

    How to read these metrics
    
              
      
                
    • Rewrites — number of rule applications (a file can count multiple times).
      • 
                
    • Files changed — distinct source files touched by at least one rewrite.
      • 
                
    • Readiness — min(100%, rewrites ÷ latentBigNumberMarkers × 100) when markers > 0 (raw ratio can exceed 100% if rewrites outnumber the coarse marker count — we cap for display).
      • 
                
    • Impact — files changed ÷ files scanned among TS/JS sources we walk (not “% of entire monorepo” if we skip paths).
      • 
                
    • Time saved — editorial estimate: 3 minutes saved per rewrite, displayed in the most readable unit.
      • 
                
    • False positives — not auto-detected today; 0 means “none reported”.
      • 
              
    
              
    
                0 rewrites on a green run often means the repo uses viem only (e.g. wagmi v2), JSBI instead of ethers.BigNumber (e.g. Uniswap v3-sdk), or @ethersproject/* patterns we do not rewrite yet — hover the warning on the rewrites cell for a short checklist.
                Some repos scan many files but match few strings (e.g. Solidity-heavy trees) — low Impact is expected until deeper rules land.
                Failed runs show Failed with a tooltip on the row via the browser’s title attribute on the status cell.
